# Campsite-Commander-
This app is used to store camping gear, food supplies, shelter.
<h3 align="center">Design Considerations</h3>

<h4> The app was designed to be simple, engaging, and practical for outdoor adventures use. </h4>

<ol>User Interface: A button field allows the user to enter the list of item used for camping.</ol>
<ol>Logic: Conditional in Kotlin to calculate the item, categorise them, also list the quantities of each item and put them in groups.</ol>

<ol>Error Handling: Invalid inputs trigger constructive feedback messages to guide the user.</ol>

<h4>Use of GitHub </h4>

Version control was managed through GitHub to ensure collaboration and progress tracking.
<ol>A repository was created to store the project files.</ol>
<ol>Regular commits documented changes and improvements.</ol>
<ol>GitHub provided a central place to manage the source code and maintain project history.</ol>

<h4> Use of GitHub Actions </h4>

GitHub Actions was implemented to automate builds and improve reliability.
<ol>A workflow file (build.yml) was set up to compile the app automatically.</ol>
<ol>Automated builds confirmed that the project runs successfully across different environments.</ol>
<ol>Logs generated by the workflow supported debugging and validation.</ol>
<ol>This setup ensured that the app was robust, maintainable, and not dependent only on local testing.</ol>
